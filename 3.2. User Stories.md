## **3.2. User Stories**

|Epic / Story ID|Título|Descripción|Criterios de Aceptación|
| :-: | :-: | :-: | :-: |
|US01|Registro de Estudiantes en la Plataforma|Como estudiante, quiero poder registrarme en la plataforma para acceder a los servicios que ofrece|<p>Escenario “El estudiante quiere crear una cuenta”: Dado que el usuario se encuentra en la pantalla de inicio, cuando seleccione la opción de “Nuevo estudiante”, entonces mediante una interfaz sencilla podrá registrar los datos que necesita para una cuenta</p><p>Escenario “El estudiante recibe un mensaje de confirmación”:</p><p>Dado que todos los datos se hayan llenado correctamente, cuando se elija la opción de “Confirmar registro”, entonces se desplegará un mensaje de confirmación de cuenta creada en la pantalla y como correo electrónico</p>|
|US02|Contacto Directo con Especialistas|Como estudiante, quiero tener contacto directo con especialistas para consultar sobre las carreras presentadas en la plataforma|Escenario: “El estudiante busca asesoría”: Dado que el usuario registrado navega por la plataforma en una sección dedicada a los académicos asociados, cuando se encuentre con las opciones de contacto de la aplicación y las selecciones, entonces desplegará tanto el correo como el número telefónico de los mismos.|
|US03|Información Detallada de Carreras|Como estudiante, quiero tener información detallada y completa sobre las diferentes carreras para tomar decisiones bien informadas|<p>Escenario “El estudiante revisa las carreras disponibles”: Dado que el estudiante navegue por la plataforma, diversos anuncios se desplegarán a lo largo presentando las diferentes carreras presentes en nuestro catálogo</p><p> </p><p>Escenario “El estudiante elige ver una carrera del catálogo”:</p><p>Dado que el estudiante elija una opción de carrera, cuando elija accionar la</p><p>opción “Más detalles”, entonces se mostrará en pantalla un resumen en general</p><p>de la carrera y una opción para tomar su respectivo test de aptitud</p>|
|US04|Tests Vocacionales para Aptitudes de Carreras|Como estudiante, quiero realizar tests vocacionales para ver a detalle las aptitudes que poseen los que toman estas carreras|<p>` `Escenario “El estudiante busca test de aptitud”: Dado que el estudiante revise el menú de opciones, cuando visualice la sección dedicada a los test de aptitudes para diversas carreras, entonces podrá tener la opción de elegir uno y realizarlo como ensayo para el futuro</p><p>Escenario “El estudiante desarrolla un test de aptitud”:</p><p>Dado que el estudiante elija realizar alguno de los test de aptitud disponibles,</p><p>cuando uno de estos sea seleccionado, entonces se procederá a dar las</p><p>preguntas correspondientes a esa prueba y las opciones para llenarla</p><p>Escenario</p><p>“El estudiante completa un test de aptitud”: Dado que el estudiante haya</p><p>llenado satisfactoriamente todas las preguntas presentadas, cuando se elija</p><p>la opción final de enviar respuestas y se confirmen sus elecciones, entonces</p><p>se enviarán sus resultados a alguno de los profesionales relacionados con la carrera elegida</p>|
|US05|Aplicación Gratuita|` `Como estudiante quiero una aplicación que sea gratuita, para no gastar|<p>Escenario</p><p>“El estudiante quiere una aplicación sin ningún tipo de gasto para usarlo”.</p><p>Dado que el usuario cuenta con una versión premium,</p><p>Cuando el usuario quiera acceder a esta, </p><p>Entonces se le solicitará una forma de pago, con esto podrá visualizar los métodos de pago disponibles.</p>|
|US06|Ahorro de Espacio en la App|Como estudiante quiero una app que no ocupe mucho espacio para no llenar mi memoria|<p>Escenario: “El estudiante busca una app que no ocupe mucho espacio”</p><p>Dado que el sistema tiene la opción de liberar un espacio de memoria</p><p>Y el usuario se dirige a Ajustes</p><p>Cuando el usuario hace click en el botón “Borrar Caché”</p><p>Entonces el sistema libera un limitado espacio de memoria para la fluidez de la aplicación</p>|
|US07|Aplicación Multiplataforma|Como estudiante quiero que la aplicación sea multiplataforma para poder usarla sin importar cuál OS tiene mi celular.|<p>Escenario: “Usuario logra acceder a la APP desde varias plataformas”</p><p>Dado que el usuario tenga una cuenta guardada en la aplicación</p><p>` `Y el usuario se dirige a Agregar Dispositivo</p><p>Cuando el usuario hace click en el botón “Agregar otro dispositivo”</p><p>Y el usuario escanea el código QR del dispositivo a agregar con la aplicación actual</p><p>Entonces el sistema guarda los dispositivos enlazados en la misma cuenta</p>|
|US08|Afiliación de Tarjeta para Pagos|Como usuario quiero afiliar una tarjeta para poder pagar las citas reservadas con los especialistas|<p>Escenario 1: El usuario afiliar una tarjeta válida a su cuenta.</p><p>Dado que el usuario cuenta con una cuenta en la app y está en la sección de afiliar una tarjeta, luego de rellenar los formularios con sus datos cuando haga clic en el botón de afiliar, entonces el sistema le mostrará un mensaje “afiliación exitosa”.</p><p>Escenario 2: El cliente trata de afiliar una tarjeta no válida.</p><p>Dado que el cliente se encuentra en la sección de afiliar tarjeta y llena todos los datos, pero con una tarjeta inválida, cuando presione el botón de afiliar, entonces el sistema le mostrará un mensaje indicando que “los datos de la tarjeta son erróneos”.</p><p>Escenario 3: El cliente no rellena de forma adecuada o deja en blanco algunos datos requeridos.</p><p>Dado que el cliente se encuentre en la sección de afiliar tarjeta y no rellena de forma adecuada o deja casilleros en blanco, cuando oprima el botón de afiliar, entonces el sistema le devolverá el mensaje “debe rellenar los datos adecuadamente y no dejar casilleros en blanco”.</p>|
|US09|Interfaz Intuitiva para Usuarios|Como usuario quiero que la aplicación tenga una interfaz intuitiva o amigable para no tener complicaciones a la hora de darle uso|<p>Escenario: El usuario experimenta por primera vez la app.</p><p>Dado que el usuario comienza a ver las interfaces de la aplicación, le es fácil manejar la app al tener interfaces amigables.</p>|
|US10|Notificaciones de Citas|Como usuario quiero que se me notifique como recordatorio las fechas de mis reuniones acordadas con un especialista para evitar ingresar tarde.|<p>Escenario: El usuario recibe una notificación de su cita.</p><p>Dado que el usuario ha reservado una reunión con un especialista, cuando llegue el día de la reunión, entonces la aplicación le enviará notificaciones para recordarle su cita.</p>|
|US11|Agenda de Citas para Especialistas|Como especialista quiero poder agendar de forma ordenada las citas en las que soy contratado para evitar retrasos y confusiones.|Escenario: El usuario selecciona la opción de agenda integrada en la aplicación. Dado que el especialista recibe múltiples citatorios con más de un usuario suscrito, cuando se selecciona esta opción él podrá ordenarlas a su gusto en una línea de tiempo y entonces podrá notificar a sus solicitantes cuando se encuentre disponible.|
|US12|Seguimiento de Aptitudes de Estudiantes|Como especialista quiero estar al tanto de las aptitudes de los estudiantes suscritos.|Escenario: El especialista verifica la información de perfil. Dado que un estudiante se haya integrado a una entrevista con un usuario especialista, cuando esté seleccione la foto del usuario en cuestión o la opción de datos, entonces se desplegará una lista detallada con los datos personales y resultados de test previos hechos por ese usuario en concreto.|
